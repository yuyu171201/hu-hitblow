# タスク2: 答えを決め合う（対戦モード）

worksheetの補足: 「ページ遷移として実装？」

## やりたいこと

PCが答えを作るのではなく、**プレイヤー同士が相手の答えを決め合って**当て合う対戦形式。
1台の端末を交代で使う「パス＆プレイ」方式を、**ページ遷移**で実現する。

想定フロー（2人プレイ）:

```
① プレイヤーAが「プレイヤーBが当てる答え」を入力
        ↓ ページ遷移
② プレイヤーBが「プレイヤーAが当てる答え」を入力
        ↓ ページ遷移
③ プレイヤーAが B の答えに挑戦（既存のguess画面を流用）
        ↓ クリアしたらページ遷移
④ プレイヤーBが A の答えに挑戦
        ↓
⑤ 結果画面（回数・時間などで勝敗）
```

## 設計メモ（着手前に決めること）

- [ ] **2-0. モードの入口をどうするか決める**
  - トップに「1人プレイ / 対戦」の選択を置く、or 対戦専用URL（例 `/vs`）を用意
  - 既存の1人プレイを壊さないこと（`/` はそのまま残す）
- [ ] **画面設計を紙orメモで確定**（上の①〜⑤に対応するページ）
  - 答え入力画面（相手に見えないよう「渡してね」画面を挟むか検討）
  - 交代の合図画面（「Bさんに代わってください」）
  - 挑戦画面（既存のindex.htmlを流用できる）
  - 結果画面

## 段階的にやること

### ステップA: サーバ側（routes.py）

- [ ] **A-1. 対戦用の状態をセッションに追加**
  - `session["mode"] = "vs"`
  - `session["secret_for_A"]`, `session["secret_for_B"]`（各プレイヤーが当てる答え）
  - `session["turn"]`（今どのフェーズか: setupA / setupB / playA / playB / result）
  - `session["tries_A"]`, `session["tries_B"]` など、既存のtriesを流用/拡張
- [ ] **A-2. 答え入力エンドポイントを追加**
  - 例 `POST /vs/set_secret` … 入力された答えをバリデーション（桁数・数字・重複なし）してセッションへ保存
  - バリデーションは既存 `routes.py` の `/guess` にあるチェックを流用
- [ ] **A-3. フェーズ遷移を管理**
  - 答えを2人分そろえたら `turn` を playA に進める
  - 既存 `/guess` を対戦にも対応させる（現在のturnに応じて参照するsecretを切り替え）
    - もしくは `/vs/guess` を別途作る（既存を壊さないなら別エンドポイントが安全）
- [ ] **A-4. 結果判定**
  - 両者クリア後、`tries`（＋余裕があれば時間）で勝敗を出す
  - b246746のスコア機能と噛み合うので、勝敗基準はチームと相談

### ステップB: 画面（templates）

- [ ] **B-4. 挑戦ページ**（既存index.htmlを流用、対戦用に見出しだけ変える）
- [ ] **B-5. 結果ページ**（両者の回数・勝敗を表示、「もう一回」ボタン）
